## БЛОК 2: Система аутентификации - Backend API

### [ФОТО КОДА]

```python
# backend/app/api/routes/auth.py
from fastapi import APIRouter, Depends, HTTPException, Request
from fastapi import status
from sqlalchemy.orm import Session
from slowapi import Limiter
from slowapi.util import get_remote_address

from app.core.database import get_db
from app.core.security import get_password_hash, verify_password, create_access_token
from app.models.user import User
from app.schemas.auth import UserLogin, UserRegister, Token
from app.services.activity_log import log_activity

router = APIRouter(prefix="/auth", tags=["auth"])
limiter = Limiter(key_func=get_remote_address)

@router.post("/register", response_model=RegisterResponse)
@limiter.limit("5/minute")
def register(request: Request, data: UserRegister, db: Session = Depends(get_db)):
    # Проверка существования пользователя
    if db.query(User).filter(User.email == data.email).first():
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="Пользователь с таким email уже зарегистрирован",
        )
    
    # Создание нового пользователя
    user = User(
        email=data.email,
        password_hash=get_password_hash(data.password),
        full_name=data.full_name,
        role=data.role,
    )
    db.add(user)
    db.commit()
    db.refresh(user)
    
    # Генерация JWT токена
    token = create_access_token(data={
        "sub": str(user.id), 
        "email": user.email, 
        "role": user.role
    })
    
    user_data = UserResponse.model_validate(user).model_dump()
    return RegisterResponse(user=user_data, access_token=token)

@router.post("/login", response_model=Token)
@limiter.limit("5/minute")
def login(request: Request, data: UserLogin, db: Session = Depends(get_db)):
    user = db.query(User).filter(User.email == data.email).first()
    
    if not user or not verify_password(data.password, user.password_hash):
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Неверный email или пароль",
        )
    
    # Логирование активности
    log_activity(db, user.id, "login", "user", user.id, {"email": user.email})
    db.commit()
    
    # Генерация JWT токена
    token = create_access_token(data={
        "sub": str(user.id), 
        "email": user.email, 
        "role": user.role
    })
    return Token(access_token=token)
```

### ОБЪЯСНЕНИЕ КОДА

Данный модуль реализует систему аутентификации и регистрации пользователей на платформе. Это критически важный компонент безопасности приложения.

**Архитектура безопасности:**

1. **Хеширование паролей**: Используется библиотека `bcrypt` через функцию `get_password_hash()`. Пароли никогда не хранятся в открытом виде, что защищает данные пользователей даже при утечке базы данных.

2. **JWT токены**: После успешной регистрации или входа генерируется JSON Web Token, который содержит:
   - `sub` (subject) - ID пользователя
   - `email` - электронная почта пользователя
   - `role` - роль пользователя (student, teacher, admin и т.д.)
   
   Токен имеет ограниченный срок действия и используется для авторизации всех последующих запросов.

3. **Rate Limiting**: Применяется ограничение скорости запросов (5 запросов в минуту) для защиты от:
   - Брутфорс атак на пароли
   - DDoS атак
   - Злоупотребления API

4. **Валидация данных**: Используются Pydantic схемы (`UserLogin`, `UserRegister`) для автоматической валидации входящих данных:
   - Проверка формата email
   - Проверка минимальной длины пароля
   - Проверка обязательных полей

5. **Логирование активности**: Все попытки входа логируются в базу данных для:
   - Анализа безопасности
   - Отслеживания подозрительной активности
   - Аудита доступа к системе

**Процесс регистрации:**
1. Проверка уникальности email
2. Хеширование пароля
3. Создание записи пользователя в БД
4. Генерация JWT токена
5. Возврат данных пользователя и токена

**Процесс входа:**
1. Поиск пользователя по email
2. Проверка пароля через `verify_password()`
3. Логирование успешного входа
4. Генерация нового JWT токена
5. Возврат токена клиенту

## БЛОК 5: API статистики дашборда - Backend расчет прогресса

### [ФОТО КОДА]

```python
# backend/app/api/routes/dashboard.py
@router.get("/stats")
def get_dashboard_stats(
    db: Annotated[Session, Depends(get_db)],
    current_user: Annotated[User, Depends(get_current_user)],
):
    """Статистика для дашборда: курсов завершено, очки, общий прогресс %."""
    # Получение всех записей студента на курсы
    enrollments = db.query(CourseEnrollment).filter(
        CourseEnrollment.user_id == current_user.id
    ).all()
    course_ids = [e.course_id for e in enrollments]
    
    if not course_ids:
        return {
            "courses_completed": 0,
            "points": current_user.points or 0,
            "progress_percent": 0,
            "total_courses": 0,
        }
    
    # Получение информации о курсах
    courses = db.query(Course).filter(Course.id.in_(course_ids)).all()
    
    # Подсчет количества тем в каждом курсе
    topics_by_course = {}
    for c in courses:
        topics = db.query(CourseTopic).filter(
            CourseTopic.course_id == c.id
        ).all()
        topics_by_course[c.id] = len(topics)
    
    # Подсчет завершенных курсов через сертификаты
    certs = db.query(Certificate).filter(
        Certificate.user_id == current_user.id,
        Certificate.course_id.in_(course_ids),
    ).all()
    courses_completed = len(certs)
    
    # Подсчет общего количества тем
    total_topics = sum(topics_by_course.values())
    
    # Подсчет завершенных тем
    progress_rows = db.query(StudentProgress).filter(
        StudentProgress.user_id == current_user.id,
        StudentProgress.course_id.in_(course_ids),
        StudentProgress.is_completed == True,
    ).all()
    completed_topics = len({p.topic_id for p in progress_rows if p.topic_id})
    
    # Расчет процента прогресса
    progress_percent = round(
        (completed_topics / total_topics * 100) if total_topics else 0, 
        1
    )
    
    return {
        "courses_completed": courses_completed,
        "points": current_user.points or 0,
        "progress_percent": progress_percent,
        "total_courses": len(course_ids),
    }
```

### ОБЪЯСНЕНИЕ КОДА

Данный endpoint реализует расчет статистики прогресса обучения студента для отображения на дашборде.

**Алгоритм расчета:**

1. **Получение записей на курсы**: Сначала получаются все записи студента на курсы из таблицы `CourseEnrollment`. Это определяет, на какие курсы студент записан.

2. **Обработка пустого случая**: Если у студента нет записей на курсы, возвращается нулевая статистика. Это предотвращает ошибки деления на ноль и обеспечивает корректную работу для новых пользователей.

3. **Подсчет тем в курсах**: Для каждого курса подсчитывается количество тем через запрос к таблице `CourseTopic`. Результат сохраняется в словаре для последующих вычислений.

4. **Определение завершенных курсов**: Завершенным считается курс, для которого студент получил сертификат. Это более строгий критерий, чем просто прохождение всех тем.

5. **Расчет общего прогресса**: 
   - Подсчитывается общее количество уникальных завершенных тем
   - Вычисляется процент от общего количества тем
   - Округление до одного знака после запятой для читаемости

**Оптимизация запросов:**
- Используется `IN` оператор для получения нескольких записей одним запросом
- Применяется `set` для удаления дубликатов завершенных тем
- Минимизировано количество обращений к базе данных

**Безопасность:**
- Используется зависимость `get_current_user` для получения авторизованного пользователя
- Все запросы фильтруются по `user_id`, что предотвращает доступ к чужим данным
- Нет возможности получить статистику другого пользователя

export function DailyQuestWidget() {
  const { t } = useLanguage();

    } finally {
      setClaiming(null);
