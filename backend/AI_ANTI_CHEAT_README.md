# Система защиты от списывания через ИИ-помощника

## Описание

Реализована комплексная система ограничений для предотвращения использования ИИ-помощника студентами для получения ответов на тесты и задания.

## Реализованные меры защиты

### 1. Улучшенный системный промпт (SYSTEM_PROMPT)

- **Строгие правила против списывания**: ИИ запрещено давать прямые ответы на тестовые вопросы, решать задания полностью, давать готовый код для заданий
- **Четкие границы**: Определено, что ИИ может делать (объяснять концепции, помогать с теорией) и что не может (давать ответы на тесты)
- **Контекстные предупреждения**: При работе в контексте теста/задания добавляются дополнительные предупреждения в промпт

### 2. Проверка подозрительных паттернов

Функция `is_suspicious_request()` проверяет запросы на наличие подозрительных фраз:
- "какой правильный ответ"
- "реши задачу"
- "напиши код для задачи"
- "выбери правильный вариант"
- И другие паттерны, указывающие на попытку получить ответ

### 3. Автоматическое определение контекста теста/задания

Функция `check_active_test_or_assignment()` автоматически определяет:
- Активные тесты студента (не пройденные)
- Активные задания студента (не сданные, с дедлайном в будущем)
- Ужесточает проверку при работе в таком контексте

### 4. Логирование подозрительных запросов

В таблице `ai_chat_history` добавлены поля:
- `is_suspicious` - флаг подозрительного запроса
- `test_id` - ID теста, если запрос был во время теста
- `assignment_id` - ID задания, если запрос был во время работы над заданием

Это позволяет администраторам отслеживать попытки списывания.

## Как это работает

1. **Студент задает вопрос ИИ-помощнику**
2. **Система проверяет контекст**:
   - Есть ли активные тесты/задания?
   - Содержит ли запрос подозрительные паттерны?
3. **Если запрос подозрительный**:
   - ИИ получает отфильтрованное сообщение или предупреждение
   - Запрос помечается как подозрительный в базе данных
   - ИИ отвечает вежливым отказом с предложением помощи по теории
4. **Если запрос нормальный**:
   - ИИ отвечает как обычно, помогая с теорией и концепциями

## Примеры защищенных запросов

❌ **Блокируется:**
- "Какой правильный ответ на вопрос: Что такое переменная? Варианты: A) ..., B) ..."
- "Реши эту задачу: написать функцию для..."
- "Дай код для задания по Python"

✅ **Разрешается:**
- "Объясни, что такое переменная в программировании"
- "Как работает цикл for в Python?"
- "Помоги разобраться с ошибкой в моем коде: [код студента]"

## Мониторинг

Администраторы могут отслеживать подозрительные запросы через запросы к таблице `ai_chat_history`:

```sql
SELECT * FROM ai_chat_history 
WHERE is_suspicious = 1 
ORDER BY created_at DESC;
```
